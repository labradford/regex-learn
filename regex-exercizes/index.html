Use regular expressions to validate this form. Assume your customers are US only. Below are some guidelines, but you can use your best judgement. This is just for fun. (I have provided a sample for the name field to get you started).

  • The name and city fields should only contain letters.
  • The street address can have both letters and numbers
  • The state should be 2 letters
  • The zip code should be 5 numbers
  • Phone format can be entered as XXX-XXX-XXXX, XXX.XXX.XXXX, or (XXX)XXX-XXXX
  • Email should have some contain the @ and . as well as the letter/numbers before, between and after
  • Credit Card should be 16 numbers starting with 5 or 4, or 15 numbers starting with 3
  • CVC Code is either 3 or 4 numbers long
  • BONUS: If the field has an error, write a message to the error div below the form field with an appropriate message
  • EXTRA BONUS CHALLENGE: use Luhn algorithm to validate credit card

Set up your user account.

USERNAMES:
  1. The only numbers in the username have to be at the end.
  2. There can be zero or more numbers at the end.
  3. Username letters can be lowercase and uppercase.
  4. Usernames have to be at least two characters long.
  5. A two-letter username can only use alphabet letter characters.
PASSWORDS:
  1. Passwords must be at least 6 characters
  2. Passwords must not be the same as the username
  3. Passwords must include at least one capital letter
  4. Passwords must include at least one lower case letter
  5. Passwords must include at least one of these characters: !@#$%^&*()+
  6. Passwords cannot include spaces
  7. No characters/letters/numbers may appear more than two times in a row
